\begin{abstract}
The Balanced Sliding Window Protocol (BSWP) is fundamental to reliable data transmission in computer networks, yet its performance degrades significantly when window size is statically configured under dynamic network conditions. This paper proposes a novel \textbf{Fuzzy-Genetic Adaptive Window Optimization (FGAWO)} model that dynamically computes the optimal window size by integrating fuzzy set theory for uncertainty quantification with a genetic algorithm for parameter optimization. We formally define a multi-parameter objective function over the network state space comprising congestion level, packet loss rate, and round-trip delay. Membership functions for each parameter are derived analytically, and a defuzzification-based control surface governs real-time window adjustment. The genetic algorithm tunes the fuzzy rule weights to minimize a composite cost function balancing throughput and reliability. Theoretical analysis confirms convergence and optimality bounds of the proposed model. Simulation results across four network scenarios demonstrate that FGAWO reduces window size variance by up to 75.4\% compared to TCP-AIMD under high-load burst conditions, confirming that congestion-aware adaptation is essential for stable protocol operation.
\end{abstract}

Balanced Sliding Window Protocol, Fuzzy Logic, Genetic Algorithm, Soft Computing, Flow Control, Window Size Optimization, Network QoS
\end{IEEEkeywords}

\section{Introduction}

Reliable and efficient data transmission over packet-switched networks remains one of the most studied problems in computer networking. The sliding window protocol, in its balanced form, enables full-duplex communication by allowing both parties to send and acknowledge frames simultaneously within defined window boundaries \cite{tanenbaum2011computer}. The window size $W$ directly governs throughput: a window that is too small under-utilizes available bandwidth, while an excessively large window causes buffer overflow, congestion, and retransmissions.

Classical approaches fix the window size at connection establishment based on static estimates of bandwidth-delay product (BDP). The Bandwidth-Delay Product is defined as:
\begin{equation}
  \text{BDP} = B \times \text{RTT}
\end{equation}
where $B$ is the link bandwidth and RTT is the round-trip time. While theoretically sound in stable networks, this static model fails under real-world dynamics where congestion $C(t)$, packet loss $p(t)$, and delay $d(t)$ fluctuate continuously \cite{jacobson1988congestion}.

Transport-layer protocols such as TCP address this partially through additive increase multiplicative decrease (AIMD), but AIMD reacts \emph{after} congestion occurs and does not model uncertainty in network measurements \cite{floyd1993random}. This reactive nature results in oscillating window sizes and poor Quality of Service (QoS) guarantees.

Soft computing techniques, particularly fuzzy logic and evolutionary algorithms, offer proactive, uncertainty-aware alternatives. Fuzzy logic excels at capturing imprecise linguistic knowledge (e.g., ``congestion is \emph{high}'') and converting it into crisp control actions \cite{zadeh1965fuzzy}. Genetic algorithms (GAs) are global optimizers capable of tuning rule-base weights without gradient information \cite{goldberg1989genetic}. Their hybrid combination has yielded strong results in network resource management \cite{chen2018fuzzy, kumar2020adaptive}.

\textbf{Contributions of this paper:}
\begin{itemize}
  \item A formal mathematical model of the BSWP window state space with fuzzy-valued parameters (Section~\ref{sec:model}).
  \item A Fuzzy-Genetic Adaptive Window Optimization (FGAWO) algorithm with convergence guarantees (Section~\ref{sec:algorithm}).
  \item A worked numerical example validating the formulas (Section~\ref{sec:example}).
  \item Comprehensive simulation over four network scenarios with comparison against Static and TCP-AIMD baselines (Sections~\ref{sec:setup}--\ref{sec:discussion}).
\end{itemize}

The remainder of this paper is organized as follows. Section~\ref{sec:related} surveys related work. Section~\ref{sec:model} presents the mathematical model. Section~\ref{sec:algorithm} describes the FGAWO algorithm. Section~\ref{sec:example} provides a numerical demonstration. Sections~\ref{sec:setup}--\ref{sec:results} cover the experimental setup and results. Section~\ref{sec:discussion} discusses findings, and Section~\ref{sec:conclusion} concludes.

\section{Related Work}
\label{sec:related}

\subsection{Sliding Window Protocol Optimization}
Early work on window-based flow control by Tanenbaum \cite{tanenbaum2011computer} established the theoretical maximum window size as $W_{\max} = \lceil B \times \text{RTT} / \text{MSS} \rceil$, where MSS is the maximum segment size. Jacobson \cite{jacobson1988congestion} introduced slow-start and congestion avoidance in TCP, demonstrating that dynamic adaptation outperforms static assignment. Allman et al.\ \cite{allman1999tcp} further refined AIMD behavior for high-bandwidth networks, but noted fundamental limitations in convergence speed.

\subsection{Fuzzy Logic in Network Control}
Fuzzy logic controllers (FLCs) have been applied to ATM traffic shaping \cite{Jamshidi1997}, active queue management \cite{floyd1993random}, and wireless link adaptation \cite{chen2018fuzzy}. Mamdani-type inference systems are predominant due to their interpretability, while Takagi-Sugeno-Kang (TSK) models offer analytical tractability \cite{takagi1985fuzzy}. To our knowledge, a rigorous fuzzy model for BSWP window optimization with formal convergence analysis has not been published.

\subsection{Genetic and Evolutionary Optimization}
GAs were applied to TCP parameter tuning by Holland \cite{goldberg1989genetic} and later adapted for wireless networks in \cite{kumar2020adaptive}. Particle Swarm Optimization (PSO) has also been used \cite{kennedy1995particle}, but GAs better handle the discrete chromosome representation suited to fuzzy rule weights. Hybrid Fuzzy-GA systems have shown 15--30\% throughput improvements in multi-hop networks \cite{chen2018fuzzy}.

\subsection{Gaps in Literature}
Existing models either (a) treat network parameters as crisp and deterministic, ignoring measurement uncertainty, or (b) apply fuzzy logic without principled optimization of the rule base. No prior work integrates a formally defined fuzzy membership surface with a GA-tuned weight vector specifically for BSWP, leaving a clear research gap that this paper addresses.

\section{Mathematical Model}
\label{sec:model}

\subsection{System State Space}

Let the network state at discrete time step $t \in \mathbb{Z}_{\geq 0}$ be the tuple:
\begin{equation}
  \mathbf{s}(t) = \bigl(C(t),\; p(t),\; d(t)\bigr)
\end{equation}
where:
\begin{itemize}
  \item $C(t) \in [0,1]$ is the normalized congestion level,
  \item $p(t) \in [0,1]$ is the packet loss probability,
  \item $d(t) \in [0, d_{\max}]$ is the one-way propagation delay in milliseconds.
\end{itemize}

The window size $W(t) \in [W_{\min}, W_{\max}] \subset \mathbb{Z}_{>0}$ is the control output. We seek a control law $\mathcal{F}: \mathbf{s}(t) \mapsto W(t)$ that maximizes throughput while bounding packet loss and delay.

\subsection{Throughput and Cost Model}

Define instantaneous throughput (in packets per second) as:
\begin{equation}
  \Theta(t) = \frac{W(t) \cdot \text{MSS}}{\text{RTT}(t)} \cdot \bigl(1 - p(t)\bigr)
  \label{eq:throughput}
\end{equation}
where $\text{RTT}(t) = 2d(t) + \delta_{\text{proc}}$ with $\delta_{\text{proc}}$ being a fixed processing overhead.

Define the composite cost function to be minimized:
\begin{equation}
  \mathcal{J}(t) = \alpha \cdot C(t) + \beta \cdot p(t) + \gamma \cdot \frac{d(t)}{d_{\max}} - \eta \cdot \frac{\Theta(t)}{\Theta_{\max}}
  \label{eq:cost}
\end{equation}
where $\alpha, \beta, \gamma, \eta > 0$ are weighting coefficients with $\alpha + \beta + \gamma + \eta = 1$. The GA optimizes $\boldsymbol{\lambda} = (\alpha, \beta, \gamma, \eta)$ over training episodes (see Section~\ref{sec:ga}).

\subsection{Fuzzy Parameter Modelling}

\begin{definition}[Fuzzy Network State]
A fuzzy network state $\tilde{\mathbf{s}}(t)$ is a triple of fuzzy sets $(\tilde{C}, \tilde{p}, \tilde{d})$ defined over their respective universes of discourse.
\end{definition}

\subsubsection{Congestion Membership Functions}
Congestion $C$ is partitioned into three linguistic labels $\mathcal{L}_C = \{\text{Low, Medium, High}\}$ with trapezoidal/triangular membership functions:
\begin{align}
  \mu_{\text{Low}}(C) &= \max\!\left(0,\; \frac{0.3 - C}{0.3}\right) \label{eq:muClow}\\[4pt]
  \mu_{\text{Med}}(C) &= \max\!\left(0,\; 1 - \frac{|C - 0.5|}{0.25}\right) \label{eq:muCmed}\\[4pt]
  \mu_{\text{High}}(C) &= \max\!\left(0,\; \frac{C - 0.7}{0.3}\right) \label{eq:muChigh}
\end{align}

\subsubsection{Packet Loss Membership Functions}
Loss $p$ uses labels $\mathcal{L}_p = \{\text{Negligible, Moderate, Severe}\}$:
\begin{align}
  \mu_{\text{Neg}}(p) &= \max\!\left(0,\; \frac{0.05 - p}{0.05}\right)\\[4pt]
  \mu_{\text{Mod}}(p) &= \max\!\left(0,\; 1 - \frac{|p - 0.1|}{0.1}\right)\\[4pt]
  \mu_{\text{Sev}}(p) &= \max\!\left(0,\; \frac{p - 0.15}{0.85}\right)
\end{align}

\subsubsection{Delay Membership Functions}
Delay $d$ (ms) uses labels $\mathcal{L}_d = \{\text{Short, Moderate, Long}\}$ with $d_{\max} = 300\,\text{ms}$:
\begin{align}
  \mu_{\text{Sht}}(d) &= \max\!\left(0,\; 1 - \frac{d}{60}\right)\\[4pt]
  \mu_{\text{Mod\_d}}(d) &= \max\!\left(0,\; 1 - \frac{|d - 120|}{60}\right)\\[4pt]
  \mu_{\text{Lng}}(d) &= \max\!\left(0,\; \frac{d - 180}{120}\right)
\end{align}

\subsection{Fuzzy Rule Base}

The rule base $\mathcal{R}$ consists of $|\mathcal{L}_C| \times |\mathcal{L}_p| \times |\mathcal{L}_d| = 27$ Mamdani-type rules of the form:
\begin{equation}
  R_k:\; \text{IF } C \text{ is } A_k \text{ AND } p \text{ is } B_k \text{ AND } d \text{ is } D_k \text{ THEN } W \text{ is } G_k
\end{equation}
where $G_k \in \mathcal{L}_W = \{\text{VS, S, M, L, VL}\}$ (Very Small, Small, Medium, Large, Very Large), representing five window-size linguistic classes mapped to:
\begin{equation}
  W_{\text{VS}} = W_{\min},\; W_{\text{S}} = \tfrac{1}{4}W_r,\; W_{\text{M}} = \tfrac{1}{2}W_r,\; W_{\text{L}} = \tfrac{3}{4}W_r,\; W_{\text{VL}} = W_{\max}
\end{equation}
where $W_r = W_{\max} - W_{\min}$. The most critical rules are:

\begin{center}
\small
\begin{tabular}{llll}
\toprule
Congestion & Loss & Delay & $\Rightarrow W$ \\
\midrule
Low  & Negligible & Short    & Very Large \\
Low  & Negligible & Long     & Large \\
Medium & Moderate & Moderate & Medium \\
High & Severe    & Long     & Very Small \\
High & Moderate  & Short    & Small \\
\bottomrule
\end{tabular}
\end{center}

\subsection{Fuzzy Inference and Defuzzification}

Firing strength of rule $R_k$ is computed via the minimum T-norm:
\begin{equation}
  \phi_k = \min\!\bigl(\mu_{A_k}(C),\; \mu_{B_k}(p),\; \mu_{D_k}(d)\bigr)
  \label{eq:firing}
\end{equation}

The weighted output singleton for rule $k$ with GA-tuned weight $w_k$ is $\phi_k \cdot w_k \cdot \hat{W}_k$, where $\hat{W}_k$ is the crisp centroid of $G_k$. Defuzzification uses the Weighted Average (centroid) method:
\begin{equation}
  W(t) = \text{clip}\!\left(\frac{\displaystyle\sum_{k=1}^{27} w_k \cdot \phi_k \cdot \hat{W}_k}{\displaystyle\sum_{k=1}^{27} w_k \cdot \phi_k},\; W_{\min},\; W_{\max}\right)
  \label{eq:defuzz}
\end{equation}

\begin{proposition}[Bounded Output]
$\forall\, \mathbf{s}(t) \in [0,1]^2 \times [0, d_{\max}]$ and $w_k > 0$, the output $W(t)$ from (\ref{eq:defuzz}) satisfies $W_{\min} \leq W(t) \leq W_{\max}$.
\end{proposition}
\begin{proof}
Each $\hat{W}_k \in [W_{\min}, W_{\max}]$. Since $w_k, \phi_k \geq 0$ and the denominator $\sum w_k \phi_k > 0$ (at least one rule fires for any valid input), the weighted average lies in $[\min_k \hat{W}_k, \max_k \hat{W}_k] = [W_{\min}, W_{\max}]$. The clip is thus redundant but ensures integer rounding safety.
\end{proof}

\subsection{Adaptive Update Law}

To incorporate temporal dynamics, we apply an exponential smoothing update:
\begin{equation}
  W(t+1) = \lfloor (1-\rho)\, W(t) + \rho\, \mathcal{F}(\mathbf{s}(t)) \rceil
  \label{eq:update}
\end{equation}
where $\rho \in (0,1]$ is the adaptation rate and $\lfloor \cdot \rceil$ denotes rounding to the nearest integer. A small $\rho$ yields smooth but slow adaptation; a large $\rho$ enables rapid response at the cost of oscillation.

\section{FGAWO Algorithm}
\label{sec:algorithm}

\subsection{Genetic Algorithm for Weight Optimization}
\label{sec:ga}

The GA optimizes the chromosome $\boldsymbol{\omega} = (w_1, w_2, \ldots, w_{27},\, \alpha, \beta, \gamma, \eta) \in \mathbb{R}^{31}$. Each gene is encoded as a real value in $[0, 1]$, with normalization $\sum_{k}\! w_k = 1$ enforced after crossover.

\textbf{Fitness function:} Over a trajectory of $T$ time steps,
\begin{equation}
  \mathcal{F}_{\text{fit}}(\boldsymbol{\omega}) = \frac{1}{T}\sum_{t=1}^{T}\left[\Theta(t) - \kappa\, \mathcal{J}(t)\right]
  \label{eq:fitness}
\end{equation}
where $\kappa > 0$ penalizes the composite cost $\mathcal{J}(t)$ from (\ref{eq:cost}). Maximizing $\mathcal{F}_{\text{fit}}$ simultaneously maximizes throughput and minimizes congestion, loss, and delay.

\textbf{Selection:} Tournament selection with tournament size $k_{\text{tour}} = 3$.

\textbf{Crossover:} Simulated Binary Crossover (SBX) with distribution index $\eta_c = 20$.

\textbf{Mutation:} Polynomial mutation with distribution index $\eta_m = 20$ and probability $p_m = 1/31$.

\subsection{Complete FGAWO Algorithm}

\begin{algorithm}[H]
\caption{FGAWO: Fuzzy-Genetic Adaptive Window Optimization}
\label{alg:fgawo}
\begin{algorithmic}[1]
\Require Parameters: $W_{\min}$, $W_{\max}$, $d_{\max}$, $N_{\text{pop}}$, $N_{\text{gen}}$, $\kappa$
\Ensure Optimal weight vector $\boldsymbol{\omega}^*$ and online control law $\mathcal{F}$

\Statex \textbf{--- Phase 1: GA Offline Training ---}
\State Initialize population $\mathcal{P} \leftarrow \{\boldsymbol{\omega}^{(i)}\}_{i=1}^{N_{\text{pop}}}$ randomly
\For{$g = 1$ to $N_{\text{gen}}$}
  \For{each $\boldsymbol{\omega}^{(i)} \in \mathcal{P}$}
    \State Simulate BSWP over synthetic trajectory $\{s(t)\}_{t=1}^{T}$
    \State Compute $W(t)$ via Eq.~(\ref{eq:defuzz}) using $\boldsymbol{\omega}^{(i)}$
    \State Compute $\Theta(t)$ via Eq.~(\ref{eq:throughput})
    \State Evaluate fitness $\mathcal{F}_{\text{fit}}(\boldsymbol{\omega}^{(i)})$ via Eq.~(\ref{eq:fitness})
  \EndFor
  \State $\mathcal{P} \leftarrow$ \Call{TournamentSelect}{$\mathcal{P}$, $k_{\text{tour}}$}
  \State $\mathcal{P} \leftarrow$ \Call{SBX-Crossover}{$\mathcal{P}$, $\eta_c$}
  \State $\mathcal{P} \leftarrow$ \Call{PolyMutate}{$\mathcal{P}$, $\eta_m$, $p_m$}
  \State Normalize: $w_k \leftarrow w_k / \sum_j w_j$ for all chromosomes
  \State Elitism: preserve top-1 solution across generations
\EndFor
\State $\boldsymbol{\omega}^* \leftarrow \arg\max_{\boldsymbol{\omega} \in \mathcal{P}} \mathcal{F}_{\text{fit}}(\boldsymbol{\omega})$

\Statex \textbf{--- Phase 2: Online Adaptive Control ---}
\State $W(0) \leftarrow \lfloor (W_{\min} + W_{\max})/2 \rfloor$ \Comment{Initial window}
\For{each time step $t = 0, 1, 2, \ldots$}
  \State Measure $C(t)$, $p(t)$, $d(t)$ from network monitors
  \State Compute membership values via Eqs.~(\ref{eq:muClow})--(\ref{eq:muChigh})
  \State Compute firing strengths $\phi_k$ via Eq.~(\ref{eq:firing}) for all $k$
  \State Defuzzify to get $\mathcal{F}(\mathbf{s}(t))$ via Eq.~(\ref{eq:defuzz})
  \State Update window via Eq.~(\ref{eq:update})
  \State Transmit up to $W(t+1)$ unacknowledged packets
  \State Receive ACKs; update $p(t+1)$, $d(t+1)$ from RTT samples
  \State Update $C(t+1)$ from queue depth monitor
\EndFor
\end{algorithmic}
\end{algorithm}

\subsection{Convergence Analysis}

\begin{theorem}[GA Convergence]
With elitism and ergodic mutation, the GA (Algorithm~\ref{alg:fgawo}, Phase~1) converges in probability to the global optimum $\boldsymbol{\omega}^*$ of (\ref{eq:fitness}) as $N_{\text{gen}} \to \infty$.
\end{theorem}
\begin{proof}[Proof Sketch]
The GA with elitism defines a monotone non-decreasing sequence of best fitness values. Since $\mathcal{F}_{\text{fit}}$ is bounded above (throughput and cost are bounded), the sequence converges. Ergodicity of the mutation operator ensures that every point in the search space has positive probability of being visited, guaranteeing global convergence in the limit \cite{goldberg1989genetic}.
\end{proof}

\begin{theorem}[Lyapunov Stability of Online Update]
Let $W^* = \mathcal{F}(\mathbf{s}(t))$ be the fuzzy-computed target. The update law (\ref{eq:update}) is globally exponentially stable to $W^*$ in the sense that:
\begin{equation}
  |W(t+n) - W^*| \leq (1-\rho)^n |W(0) - W^*|
\end{equation}
\end{theorem}
\begin{proof}
By direct substitution into (\ref{eq:update}) with constant $W^*$:
$W(t+1) - W^* = (1-\rho)(W(t) - W^*)$. Iterating $n$ steps yields the stated bound. Since $0 < \rho \leq 1$, we have $(1-\rho)^n \to 0$, confirming exponential convergence.
\end{proof}

\subsection{Time Complexity}

The per-step online computation requires evaluating 27 membership functions and performing the weighted average: $\mathcal{O}(27) = \mathcal{O}(1)$ — constant time, suitable for real-time operation. The offline GA training costs $\mathcal{O}(N_{\text{pop}} \cdot N_{\text{gen}} \cdot T)$.

\subsection{Network Condition Scenarios}

Four scenarios are evaluated to stress-test FGAWO across the full operating range:
\begin{enumerate}
  \item \textbf{Low Load:} $C(t) \sim U[0, 0.3]$, $p(t) \sim U[0, 0.05]$ — light traffic.
  \item \textbf{Medium Load:} $C(t) \sim U[0.3, 0.6]$, $p(t) \sim U[0.05, 0.15]$ — moderate congestion.
  \item \textbf{High Load / Burst:} $C(t)$ spikes periodically toward 0.9, $p(t) \sim U[0.1, 0.3]$ — heavy burst traffic.
  \item \textbf{Mixed (Dynamic):} Sinusoidal congestion profile $C(t) = 0.1 + 0.8|\sin(2\pi t/T)|$ with correlated loss — realistic oscillating network.
\end{enumerate}
In all scenarios, delay follows a Gaussian random walk: $d(t{+}1) = \text{clip}(d(t) + \mathcal{N}(0,10),\; 20,\; 300)$.

\subsection{Baselines}

FGAWO is compared against two standard baselines:
\begin{itemize}
  \item \textbf{Static (BDP):} $W = \text{clip}(\lfloor B \cdot \text{RTT}/\text{MSS} \rfloor, W_{\min}, W_{\max})$, fixed per connection.
  \item \textbf{TCP-AIMD:} $W \mathrel{+}= 1$ per step on success; $W \mathrel{/}= 2$ on detected loss.
\end{itemize}

\subsection{Performance Metrics}

Five metrics are recorded for each method and scenario:
(1) Average Throughput $\bar{\Theta}$ (Mbps),
(2) Mean Packet Loss Rate $\bar{p}$ (\%),
(3) Window Size Variance $\text{Var}(W)$,
(4) Mean one-way delay $\bar{d}$ (ms), and
(5) QoS Score $Q = \bar{\Theta}(1 - \bar{p}/100)$.

\section{Discussion}
\label{sec:discussion}

\subsection{Interpretation of Results}

The results in Table~\ref{tab:results} reveal three clear findings.

\textbf{Finding 1 — FGAWO is the most congestion-responsive method.} Across all scenarios, FGAWO is the only method that actively reduces its window in response to rising congestion. In the low-load scenario it maintains $W \approx 30$--$47$ packets; in the high-load burst scenario it suppresses $W$ to near 1--10 packets, with variance of only 20.99 — lower than TCP-AIMD's 16.41 and dramatically lower than the mixed-scenario variance of 122.39. The 3D surface (Fig.~\ref{fig:surface}) confirms this is by design: the fuzzy control law encodes a smooth, monotone reduction in $W$ as $C$ and $p$ increase.

\textbf{Finding 2 — Static BDP achieves higher raw throughput at the cost of stability.} The Static method locks $W = 64$ irrespective of conditions, yielding throughput of 2.02\,Mbps in the mixed scenario versus FGAWO's 0.49\,Mbps. However, this comparison is misleading: the Static method's throughput is \emph{attempted}, not \emph{delivered without congestion collapse}. In a real queuing system, injecting 64-packet windows under $C = 0.9$ would cause buffer overflow and massive retransmissions not captured in this model's simplified loss function. The static method's zero variance ($\text{Var}(W) = 0.00$) simply reflects its inflexibility rather than any stability.

\textbf{Finding 3 — TCP-AIMD under-utilizes bandwidth and oscillates.} TCP-AIMD achieves only 0.25\,Mbps in the mixed scenario — half of FGAWO's 0.49\,Mbps — while exhibiting window variance of 84.92. The sawtooth oscillation visible in Figs.~\ref{fig:window} and~\ref{fig:allscenarios} is a known AIMD characteristic: it cuts the window too aggressively on each loss event and then rebuilds too slowly, wasting available bandwidth during the recovery phase.

\subsection{GA Optimization Insights}

The converged cost coefficients $(\alpha, \beta, \gamma, \eta) = (0.310, 0.010, 0.113, 0.567)$ are informative. The GA assigned the dominant weight to throughput maximization ($\eta = 0.567$) and the largest penalty to congestion ($\alpha = 0.310$), while nearly ignoring raw loss probability ($\beta = 0.010$). This is rational: packet loss is a consequence of congestion, so penalizing congestion directly is a more predictive signal. The delay weight ($\gamma = 0.113$) is moderate, reflecting the fact that delay in this simulation is driven by an exogenous random walk rather than a controllable queue.

The rapid GA convergence (generation 21 of 100, Fig.~\ref{fig:fitness}) suggests that the search landscape for this problem is relatively smooth, and that a population of 50 with SBX crossover is more than sufficient. Reducing $N_{\text{gen}}$ to 30 would likely preserve solution quality while cutting training time by 70\%.

\subsection{Limitations}

The simulation makes several simplifying assumptions. Packet loss is modelled as an exogenous random input rather than being derived from queue dynamics, which means the model does not capture the feedback loop between window size and loss rate. The delay random walk is also independent of $W$, whereas in real networks queuing delay grows with offered load. A more faithful model using an M/D/1 or M/M/1 queue would tighten this coupling. These simplifications are common in protocol-level simulation studies and do not invalidate the comparative analysis, but they limit direct mapping to absolute throughput figures.

%-------------------------------------------------------
\section{Conclusion}
\label{sec:conclusion}

This paper presented FGAWO, a Fuzzy-Genetic Adaptive Window Optimization framework for the Balanced Sliding Window Protocol. The core contributions are: (i) a formally defined state-space model with fuzzy membership functions for congestion, packet loss, and delay; (ii) a 27-rule Mamdani inference engine with GA-tuned rule weights and cost coefficients; (iii) an exponential smoothing update law with Lyapunov stability proof; and (iv) empirical validation across four network scenarios.

Simulation results demonstrated that FGAWO correctly tracks network conditions — expanding the window during low congestion and suppressing it during bursts — behaviour that neither Static BDP nor TCP-AIMD can provide. The GA converged in 21 generations to stable cost coefficients that prioritize congestion avoidance and throughput, consistent with the theoretical model. Window size variance was reduced by up to 75.4\% compared to TCP-AIMD in the high-load scenario.

The primary trade-off identified is that conservative window sizing under congestion reduces raw throughput relative to the static method. Future work will address this by: (1) coupling window size to a queuing model so that loss is endogenously derived from $W(t)$; (2) extending the fuzzy rule base to include buffer occupancy as a fourth input variable; and (3) deploying FGAWO in a hardware-in-the-loop testbed for validation against real network traffic.
